## Session 页结构说明

每个 session 页有四段：

1. **会议元信息**：时间、带读人、主 paper 链接、参与者
2. **Pre-read**：会前阅读材料 + 1–3 个引导问题
3. **Live notes**：会中讨论的关键点（带读人 / 记录员实时记）
4. **Post-meeting**：会后整理的 key insights + action items + 链向相关 paper page / 内部项目

模板见 `@/scripts/new-session.sh`（`pnpm new:session`），自动生成。

---

## 排程规则

- **周一上午** PI / Postdoc 选下周共读 paper（参考 [研究主线](/themes/) 的开放问题）
- **周三**之前确定带读人 + 创建 session 页（运行 `pnpm new:session 2026-W19 paper-slug`）
- **周日**之前所有人在 session 页留 pre-read 问题
- **周一**周会
- **周二**带读人补完会议纪要
